Best Practices | Classes vs. IDs

When creating a website - like the one you’re visiting right now - it is very important to know about classes and IDs first.

In a nutshell, classes and IDs allow you to apply styling in a more detailed way. They will help you to target elements that you want to edit with ease and flexibility. In HTML Classes and IDs are attributes, and in CSS they are selectors.

The differences between the two are…

ID’s use the [#] symbol and are unique! This means each element can have only one ID, because otherwise it wouldn’t be unique, right? Very importantly, each page can have only feature one element with that unique ID. This means, once you use an ID on your page you shouldn’t try to use it again. Sort of like a disposable camera. IDs are helpful when you want to specify a special design feature without affecting the rest of the content with this special update. For example, the special sidebar links that feature in this site with rotated text to frame the page like a border.

Classes use the [.] symbol and are *sadly* not unique! The upside of their ordinariness is that they’re multipurpose and reusable! You can use the same class on multiple elements, as well as multiple classes on the same element. This means that classes are hard working and have got your back! They’re very efficient and helpful when you need to style information that needs to be applied to multiple objects on a page. For example, to format the images, headings, sub headings and paragraph text of each blog post on this site.

A great analogy that I’ve read about classes vs attributes compares them to barcodes and serial numbers. Packaging for products will feature a barcode. That barcode contains and provides broad, usually universal information about the product. All products of the same type have the exact same barcode.

Products usually also have a serial number which is completely unique to any other product in the world. Unlike barcodes, serial numbers do not contain broad information about the product. They just specify details about the one product.

Following that analogy logic, information that is reusable should be kept in a class and information that is totally unique should be kept in an ID. Simple!



Pro Tips

Make sure to make your classes and IDs descriptive! It makes it a lot easier to know what you’re looking at when editing! And, always remember that Classes and ID’s don’t have any styling information to them all by themselves. They require CSS to target them and apply styling.
